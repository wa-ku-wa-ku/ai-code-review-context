# 阶段验收流程

每个阶段完成后，不要只看 Codex 的文字说明，要按下面流程验收。

## 1. 检查修改范围

确认 Codex 是否只修改了当前阶段相关文件。

重点检查：

```text
有没有提前实现后续阶段？
有没有引入无关依赖？
有没有把评审判断逻辑写进上下文模块？
有没有修改无关配置？
```

如果超范围，要求 Codex 回退或删除超出当前阶段的内容。

## 2. 运行测试

至少运行：

```bash
pytest
```

如果只验收某个模块，也可以运行：

```bash
pytest tests/test_xxx.py
```

## 3. 对照验收标准打勾

每个阶段任务文件中都有验收标准。

没有全部满足，不进入下一阶段。

## 4. 让 Codex 做阶段总结

可以让 Codex 输出：

```text
1. 阶段目标
2. 完成任务
3. 可见产物
4. 测试结果
5. 已知限制
6. 下一阶段计划
```

这个总结可以放进周报、答辩过程记录或项目管理文档。

## 5. 提交代码

建议每个阶段单独 commit：

```bash
git add .
git commit -m "feat: implement stage 1 file scanner"
```
