# 上下文处理模块阶段任务总览

## 总体目标

从 0 开发一个面向 Python 仓库级代码评审的上下文处理模块。

模块输入：Python 仓库或 zip 包。

模块处理：扫描、解析、建索引、生成任务、提供上下文工具、追踪覆盖率。

模块输出：repo_summary、review_tasks、task_card、代码查询工具、coverage_report。

## 阶段总表

| 阶段 | 阶段名称 | 可验收目标 |
|---|---|---|
| 阶段 0 | 工程骨架搭建 | 项目能启动，pytest 能运行，sample_repo 存在 |
| 阶段 1 | 仓库输入与文件扫描 | 能读取仓库并扫描有效 Python 文件 |
| 阶段 2 | AST 符号解析 | 能抽取函数、类、方法、import、decorator |
| 阶段 3 | SQLite 索引存储 | 能生成 code_files、code_nodes、code_edges |
| 阶段 4 | 调用关系与路由识别 | 能识别 FastAPI / Flask 入口和调用关系 |
| 阶段 5 | 上下文工具接口 | Agent 能调用工具查源码、符号和调用链 |
| 阶段 6 | 评审任务生成 | 能生成 repo_summary、review_tasks、task_card |
| 阶段 7 | 覆盖率追踪与验收 | 能生成 coverage_report，证明评审范围 |

## 阶段逻辑

```text
阶段 1：我能找到代码
阶段 2：我能理解代码结构
阶段 3：我能存下代码图谱
阶段 4：我能知道代码关系
阶段 5：我能把图谱能力暴露给 Agent
阶段 6：我能把仓库拆成评审任务
阶段 7：我能证明哪些代码被覆盖
```
